Textures

IImage represents a 1D-, 2D- or 3D-bitmap, either compressed or uncompressed.

Images can be stored in various pixel formats (see ImagePixelFormat). To convert images between pixel formats you may use the corresponding instance of IImageFormatConverter. Engine includes the following image convertors as modules:

  • Rgb2LuminanceImageConverter;
  • Rgba2LuminanceImageConverter.

ICore keeps a list of image format converters. You may register one using the ICore::registerImageFormatConverter method. There is also the convenient ICore::convertImageFormat method available for image conversion.

ITextureImage is a collection of faces (1 for normal textures or 6 for cube maps). Each face is represented as a collection of mipmap levels (1 or more). Each mipmap level is an instance of IImage. To load texture you may use the corresponding instance of ITextureLoader. Engine includes the following texture loaders as modules:

  • PngTextureLoader.

ICore keeps a list of texture loaders. You may register on using the ICore::registerTextureLoader method. There is also the convenient ICore::loadTextureImage method available to load textures.

Resource manager provides developer with a Texture handle. The corresponding texture may be either resident in memory or not. The ITextureSource interface has been provided for resource manager to be able to reload textures on demand.
